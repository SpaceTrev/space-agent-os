# Marketing Dependencies

## Needs from Upstream (Planning / Engineering)
- Completed feature or milestone — what shipped and when.
- Linear issue(s) closed in the release — for accurate feature description.
- Any messaging guidance or constraints from humans (e.g., "don't mention X yet").
- Target audience for the content piece.

Marketing does not invent capabilities. All claims must be verifiable from closed Linear issues or the codebase.

## Delivers Downstream (Humans for approval)
- Draft content (blog post, release notes, social copy) ready for human review.
- Suggested publish date and channel.
- A brief note on the key message and why it resonates with the audience.

## Shared Contracts
- Marketing never publishes directly. All content requires human approval before going live.
- Release notes are canonical — they live in `CHANGELOG.md` at the repo root and must match what actually shipped.
