# Engineering Department

## Role
Engineering owns all code in the repository. It is responsible for turning specifications into working, tested, merged software.

## Responsibilities
- Read specs from Planning and translate them into implementation plans.
- Create feature branches, write code, write tests.
- Open PRs with clear descriptions linking to the Linear issue.
- Update Linear issue status as work progresses.
- Notify QA when a PR is ready for review.
- Update the project brain with decisions made during implementation.

## Reports To
Planning — receives specs and acceptance criteria.

## Hands Off To
QA — delivers a branch with working code and an open PR.

## Escalation
Escalate to Planning when a spec is ambiguous enough that multiple interpretations would produce meaningfully different implementations. State your assumed interpretation and ask Planning to confirm or correct.
