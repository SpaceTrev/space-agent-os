## 22:40 — The onboarding flywheel (core product insight)

This is the full loop:

**Stage 1: Intake chatbot**
- Asks about business, industry, team size, needs
- Uses knowledge from ALL previous customer flows to inform answers
- Industry-specific research + model trained on "teams and functions by industry"
- Output: auto-configured agent team suggestion

**Stage 2: Personal assistant activation**
- Space-Claw turns on, observes how the human actually works
- Watches their tools, workflows, communication patterns
- Starts suggesting automations based on what it sees

**Stage 3: Workflow definition**
- Agent builds tools to automate observed processes
- Human approves/refines
- Workflow gets codified — defined steps, triggers, outputs

**Stage 4: Autonomous execution**
- Once workflow is well-defined, hand it to a smaller/cheaper model
- That model just executes the known process — no reasoning needed
- Orchestrator kicks it off on trigger, monitors, escalates if something breaks

**The efficiency curve:**
- Week 1: Claude (expensive, reasoning) figuring out the client's world
- Week 4: Mix of Claude + smaller models as patterns emerge  
- Month 3: 80% of recurring work running on cheap/fast models, Claude only for novel tasks
- Token costs drop dramatically as the system learns, workflows compound

**The network effect:**
- Client A in Mexican retail trains the retail brain
- Client B in retail starts with that brain + their specific onboarding
- The more clients in a vertical, the better the starting point for everyone
- Cross-client workflow templates (anonymized) compound the moat

**What this means for the build:**
- Onboarding chatbot = first thing a new client sees (not a signup form)
- "Observation mode" for space-claw = watching + suggesting, not just executing
- Workflow codification UI = how humans approve and lock in automations
- Model routing intelligence = automatically downshift to cheaper model once workflow is proven
- Industry brain = the knowledge layer that makes intake fast and accurate
