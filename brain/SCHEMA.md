# Space Scribe — Brain Document Schema

Every file in `brain/` must begin with a YAML frontmatter block. The brain
loader parses this block to build the document index, score relevance, and
respect token budgets.

---

## Frontmatter Spec

```yaml
---
id: kebab-case-unique-slug             # required — unique across all vaults
title: Human-Readable Title            # required
vault: company|engineering|marketing|sales|operations|skills   # required
domain: optional-sub-domain           # optional (e.g. python, react, auth, discord)
tags:                                  # optional — used for relevance matching
  - tag-one
  - tag-two
priority: always|high|normal|low      # required — controls loading order
  # always  → loaded in every agent session (company brain)
  # high    → loaded when vault matches agent department
  # normal  → loaded when tags match task keywords
  # low     → loaded only when explicitly requested
source: path/to/origin.md             # optional — original file this was seeded from
created: 2026-04-06                   # required — ISO date
updated: 2026-04-06                   # required — ISO date, update on every edit
token_estimate: 500                   # optional — rough char/4 estimate; loader auto-fills
---
```

---

## Vault Definitions

| Vault | Path | Audience | Priority rule |
|-------|------|----------|--------------|
| `company` | `brain/company/` | All agents | `always` docs loaded in every session |
| `engineering` | `brain/engineering/` | Engineering agents | `high` when dept=engineering |
| `marketing` | `brain/marketing/` | Marketing agents | `high` when dept=marketing |
| `sales` | `brain/sales/` | Sales agents | `high` when dept=sales |
| `operations` | `brain/operations/` | Ops agents | `high` when dept=operations |
| `skills` | `brain/skills/` | All agents | `normal` — matched by tags/domain |

## Document Body Guidelines

- **Lead with the actionable.** Agents use these as reference, not reading material.
- **No fluff.** If a sentence does not change agent behavior, cut it.
- **Use headings and tables.** Structured content tokens better than prose.
- **Link, don't duplicate.** If something is in another brain doc, reference it by `id:`.
- **Update on every task.** After a task touches a brain domain, update `updated:` date.
- **Token budget.** Target ≤ 800 tokens per doc. Split larger docs by topic.

---

## Priority Loading Rules

The assembler fills the context window in this order until the token budget is
exhausted:

1. `priority: always` — all docs (company brain, non-negotiable)
2. `priority: high` — docs in the agent's vault
3. Skills docs whose `tags` intersect the task keyword set
4. `priority: normal` docs whose `tags` intersect the task keyword set
5. Project-specific context from active task

Default token budget: **20 000 tokens** (≈ 10% of Claude Sonnet 4.6's 200k window).
Override with `BRAIN_TOKEN_BUDGET` env var.

---

## Skill Doc Conventions (`vault: skills`)

Skill docs capture reusable patterns extracted from completed tasks. They must
include:

- **Pattern** — what situation triggers this skill
- **Steps** — numbered, concrete actions
- **Example** — minimal illustrative snippet
- **Antipatterns** — what not to do

Skill docs are auto-generated by `apps/core/brain/extractor.py` after every
task completion. Human review is encouraged but not required before they load.
